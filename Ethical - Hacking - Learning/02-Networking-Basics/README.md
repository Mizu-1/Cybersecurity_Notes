## IP ADDRESS

- هو عنوان يتكون من سلسلة من الارقام لكي تحدد جهاز محدد داخل الشبكة سواء كانت شبكة محلية او شبكة انترنت

طريقة اظهار العنوان في لينكس بالامر ifconfig وفي الويندوز ipconfig

اشكال العنوان شكلين inet \ ipV4 و inet6 \ ipV6 

inet \ ipV4 يكون مكتوب بالشكل دسمل
inet6 \ ipV6 يكون مكتوب بشكل هيكسا دسمل

ipV4 هذا العنوان الذي يستخدم الى الان بينما ipV6 هو العنوان الجديد او بالشكل الجديد

> [!NOTE] inet \ ipV4  ( 000.000.000.000 )
> كل ثلاث ارقام بعد النقطة تسمى مجموعة
> ويكون من 4 مجموعات
> وكل مجموعة عبارة عن 8 بت كلها 32 بت
> وكل مجموعة تاخذ رقم مابين 0 الى 255 فقط
> 


> [!NOTE] inet6 \ ipV6 fe80::5be4:ffe9:7c82:3112
>كل اربع خانات تكون مجموعة
>ويتكون من 8 مجموعات كلها 128 بت
>كل مجموعة ممكن ان تتكون من ارقام وحروف

## MAC ADDRESS

هو عنوان بتم تخصيصة لكل جهاز بحتوي على بطاقة تسمى NIC ( Network Interface Card ) 

ومعنى MAC ( Media Access Control ) 

طريقة اظهار عنوان الماك عن طريق كتابة الامر ifconfig وفي الويندوز ipconfig

لمعرفة عنوان الماك يكتب عند ether او Physical Address

> [!NOTE] ether 00:0c:29:c0:b9:70
> 
> يتكون من 12 مجموعة 
> ومكتوب بنظام هيسكا ديسمل
> وكل مجوعة عبارة عن خانتين حروف وارقام
> 

يمكن لعنوان الماك تحديد نوع الجهاز ومن اي شركة بالضبط عن طريقة اول ثلاث خانات

> [!NOTE] ether ==00:0c:29==:c0:b9:70
> 
> تسمى OUI ( Organizationally Unique Identifer )

في موقع قوقل يسمى MAC Address Lookup تضع عنوان الماك ويظهر لك النتائج

## TCP , UDP , THREE-WAY HANDSHAKE

- **TCP** ( Transmission Control Protocol ) يعتمد على الاتصال الموجه 

ويعرف كبروتوكول اتصال موثوق ويقوم بجلسة اتصال بين الطرفين قبل تبادل البيانات وبتم التاكد من وصول البيانات دون فقدان او تكرار اي بيانات

ويعتمد الاتصال على طريقة Three-Way Handshake

الخصائص الاساسية :

- يضمن اوصل كل جزء من البيانات بالترتيب الصحيح ( High Reliability )
- الكشف عن الاخطاء ويعيد البيانات المفقودة ( Error Checking )
- يستعمل طريقة هذه ليتاكد ان الطرفين جاهزين للاتصال ( Three-Way Handshake )

تطبيقاتها :

- خاص للتصفح في الانترنت ( HTTP \ HTTPS )
- خاص لنقل الملفات ( FTP )
- خاص لادارة الاجهزة عن بعد ( SSH )

- **UDP** ( User Datagram Protocol ) بكون اتصال غير موجه

بعرف كبروتوكول غير موثوق ولا يقوم بجلسة اتصال بين الطرفين ويقوم برسال البيانات بسرعة من دون التاكد من وصولها او ترتيبها 

الخصائص الاساسية :

- سرعة عالية ولا يتحقق من صحه او اخطاء البيانات  ( Faster Transmission )
- غير موثوق بالكامل ممكن ان توصل البيانات غير مرتبة او مفقودة  ( Not Entirely Reliable )

تطبيقاتها :

- البث المباشر ( streaming )
- تحويل اسماء الدومينز او عناوين DNS ) IP )
- المكالمات الاساسية عن طريق الانترنت ( VoIP )

- **Three-Way Handshake**

اهميتها هي يضمن للطرفين مستعدين لتبادل البيانات ويساعد في اكتشاف الاخطاء المحتمله في الخطوة التالية

خطوات ارسال البيانات عن طريق TCP :

العميل Client يرسل حزمة ==SYN== => للخادم Server
ويرد الخادم Server بحزمة ==SYN-ACK== => للعميل Client 
العميل Client يرسل حزمة ==ACK== => للخادم Server

( SYN = الموافقه على بدء الاتصال )
( ACK = ياكد استلام رسالة الاتصال )

- **Ports** المنافذ

هي نقطه اتصال داخل الجهاز مستخدم التطبيقات او البروتوكولات عشان تبادل البيانات او المعلومات

كل بوتوكول بستخدم منفذ محدد 

عدد المنافذ المتاحة 65,536 لكل جهاز

المنافذ المعروفة من 0 الى 1023

والمنافذ الخاصة او الغير معروفة من 1024 الى 65,536

مثال عليها نفس المثال السابق لكن الخادم Server يكون المنفذ 
من جهازي الى HTTPS/443

العميل Client يرسل حزمة ==SYN== => ل443
ويرد ال443 بحزمة ==SYN-ACK== => للعميل Client 
العميل Client يرسل حزمة ==ACK== => ل443

> [!NOTE] **Wireshark**
> 
> هي اداء لعمل تحليل لكل حزمة حدثت امام عينك في الBackEnd
> 
> تعرض كل حزمة التي تم تبادلها في الشبكة

اهمية TCP , UDP في الامن السبراني :

كجزء في الاختراق يتم فحص المنافذ المفتوحة لمعرفة الخدمات الشغالة على الجهاز المستهدف

ويوجد ادوات تسمى TCP Scan و UDP Scan 

اداء TCP Scan هي لفحص الخدمات التي تطلب اتصال موثوق مثل HTTP

اداء UDP Scan هي لفحص الخدمات التي تعتمد سرعة الاتصال مثل DNS

مثال لها في Nmab يتم ارسال حزم لمنافذ لجهاز المستهدف اذا استقبال او استجاب الحزم يعني ان المنفذ مفتوح وفي حال لم يستقبل او بستجيب يعني المنفذ مغلق ومحمي

## COMMON PORTS & PROTOCOLS

قبل الدخول لOSI Model يجب معرفة البروتوكولات والمنافذ التي سوف تتعامل معها كمختبر امني

سبب معرفة البروتوكولات والمنافذ لانها تشكل الاساس لفهم الشبكات واختبار الامن والتي سوف تسهل عليك التعرف على الخدمات عندما تفحص جهاز او شبكة معينة وهدفها تجعلك اكثر كفاءة وسرعة في التحليل .

- **بروتوكل TCP :**

- FTP ( File Transfer Protocol ) { منفذ 21 }
نقل الملفات بين الجهاز المستخدم والخادم Server , يوفر رفع الملفات وتنزيلها من والى الخادم Srever , وظهر في اختبارات ( المسك بالعلم ) او تحليل بين الاجهزة .
ونقطة الامن انه ينقل الملفات في نص واضح غير مشفر .

- SSH ( Secure Shell ) { 22 منفذ }
- Telnet { 23 منفذ}
هما لعمل تسجيل دخول للاجهزة عن بعد 
في Telnet عيبها ان البيانات تنتقل بشكل نص واضح وتجعلها غير امنه , اما SSH يكون مشفر وامن 
الاستخدام العملي ل SSH هو الخيار الافضل لاستخدام الامن لما تدير الاجهزة عن بعد

- SMTP ( Simple Mail transfer Protocol ) { 25 منفذ }
يستخدم لارسال البريد الالكتروني

- POP3 ( POst Office Protocol ) { 110 منفذ }
هو استرجاع البريد من الخادم Server لكن لا بحافظ على النسخة التي على الخادم Server 

- IMAP ( Internet Massage Access Protocol ) { 143 منفذ }
هو يسترجع البريد من الخادم Server لكن بحفظ نسخة على الخادم Server بعكس الPOP3

{ ملاحظة IMAP - POP3 - SMTP : البروتوكلات التي مرتبطه بالبريد الالكتروني لاتكون محور التركيز في اغلب دورات الامن السبراني لكن يظهر في بعض الاختبار التي سوف تفعلها مستقبلا }

- DNS ( Domain Name System ) { 53 منفذ }
هو ترجمه Domains الى عنوان IP Address خاص بالموقع
{ ملاحظة : يجعل الانترنت اكثر سهولة للبشر }

- HTTP ( Hypertext transfer Protocol ) { 80 منفذ }
هو يستخدم لنقل الصور والنصوص على الويب , وعيبها ان البيانات لا تكون مشفرة اي غير امنة

- HTTPS ( ypertext transfer Protocol Secuer ) { 443 منفذ }
هو يستخدم لنقل الصور والنصوص على الويب , لكن مشفرة وامنه عن طريق SSL و TLS  بعكس HTTP

- SMB ( Server Message Block ) { 139 - 445 منافذ }
هو مشاركة الملفات عبر الشبكة

- **برتوكول UDP :**

- DNS  { 53 منفذ }
بشتغل على TCP و UDP 
في TCP يستخدم في الطلبات المعقدة , وفي UDP يستخدم في الطلبات البسيطة

- DHCP ( Dynamic Host Configuration Protocol ) { 67 - 68 منافذ }
هو تخصيص IP Address متحرك للاجهزة المتصلة بالشبكة

- TFTP ( Trivial File Transfer Portocol ) { 69 منفذ }
هو نقل الملفات بطريقة بسيطة باستخدام UDP
الفرق بينه وبين FTP اسرع لكن اقل امان

- SNMP ( Simple Network Mangment Protocol ) { 161 منفذ }
هو ادارة الشبكات ويجمع البيانات عنها
